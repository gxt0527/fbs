## 2026-07-10 — LiquidGlass UI 重新设计

### 背景
基于 AndroidLiquidGlass (Backdrop) 设计语言重新设计 FBS Flutter 用户界面，仅改 UI，不动功能代码。

### 设计语言来源
- GitHub: https://github.com/Kyant0/AndroidLiquidGlass
- 文档: https://kyant.gitbook.io/backdrop
- 核心效果: 背景模糊 (BackdropFilter) + 半透明渐变容器 + 微妙边框 + 胶囊圆角

### 设计令牌 (GlassTokens in main.dart)
| 令牌 | 值 | 用途 |
|------|-----|------|
| accent | #0088FF (light) / #0091FF (dark) | 强调色 |
| glassLight | #FAFAFA @40% | 浅色玻璃容器 |
| glassDark | #121212 @40% | 暗色玻璃容器 |
| 圆角 | 8/12/16/20/24/999 | 分级圆角，999=胶囊 |
| 间距 | 4/8/16/24/32 | 8dp网格系统 |
| 阴影 | black@5% blur 4 + offset(0,2) | 极微妙阴影 |
| 边框 | white@25% (light) / white@8% (dark), 0.5px | 玻璃边缘 |

### 玻璃效果实现方案 (Flutter 替代 Backdrop Compose API)
- **模糊**: `BackdropFilter` + `ImageFilter.blur(sigmaX:8, sigmaY:8)` → 代替 `blur()`
- **色彩增强**: LinearGradient 半透明叠加 → 代替 `vibrancy()`
- **玻璃边框**: 0.5px 白色半透明边框 → 代替 `InnerShadow`
- **胶囊形**: `BorderRadius.circular(999)` → 代替 `Capsule()`
- **组件封装**: `GlassSurface` Widget → 代替 `drawBackdrop()`
- **交互**: 保留主题系统 AnimatedContainer（不实现 Lens 折射/弹性变形，Flutter 不支持）

### 2026-07-10 色轮多轮迭代（HSV 色轮完整方案）
**问题演进 → 解决（每步都 build+install 到 33de4df9）:**
1. **色轮不居中/拾色错位/右侧硬边**: `Positioned(top:offset*0.6)` 偏移 hack → 改 `Center`+固定 240×240 SizedBox，中心点固定 (120,120)、有效半径 116，手势和指示器用同一坐标系；SweepGradient 首尾闭合+`0→2π`
2. **大标题/背景无法选、副标题/正文首次不跟手**: 色轮局部 hue/sat 在重建时丢失 → 提升为实例字段 `_wheelHue/_wheelSat/_wheelSyncedTarget`，切换目标自动同步；手势加 `onPanDown` 首次即响应
3. **颜色过渡有色带(banding)**: HSL 空间 L=0.5 各色相亮度不一致 → 改 **HSV 空间**，自写 `_hsvToRgb(h,s,v)` (静态方法)，绘制器用 **120 圈同心弧**逐圈精确算饱和度；拾色与绘制用同一 `_hsvToRgb` 保证所见即所得
4. **滚动卡顿**: 每次滚动重建都重绘 120 圈弧 → **预渲染为 `ui.Image` 缓存**(`_wheelImageCache` 静态字段, `PictureRecorder`+`FutureBuilder`+`RawImage`)，只算一次
5. **无法选白/灰/黑**: 色轮固定 V=1.0 只有 H+S 二维 → 加 **明度滑块**(`_wheelValue`) + `_rgbToHsv()` 提取三分量；滑块黑→全亮色 LinearGradient 背景，`_hsvToRgb(h,s,value)` 拾色
- **关键经验**: HSV 优于 HSL 做色轮(等亮度)；昂贵 CustomPainter 必须预渲染 ui.Image 缓存；色轮=H(角度)+S(半径) 二维，必须配 V 滑块才能选灰阶
