## 背屏通知转发修复（基于官方APK逆向分析）

### 分析对象
- `E:\MSRR\背屏_RELEASE-1.0.2605272226.apk` (19MB, com.xiaomi.subscreencenter)
- 使用 aapt + dexdump 提取了 Manifest、全部类名、36255 个字符串常量

### 核心问题
FBS 原有 `displayOnBackScreen()` 使用了三个虚构的接口：
1. `SubScreenActivity` — APK中不存在
2. `SHOW_NOTIFICATION` 广播 — APK中不存在
3. `content://...provider/notification` — APK中不存在

### 官方背屏实际接口
1. **PinReceiveActivity** (`ACTION_SEND`, text/* + image/*) — 唯一公开投屏入口
2. **notification_widget.json** — 通知数据存储
3. **miui.intent.action.SUB_SCREEN_ON/OFF** — 背屏开关广播
4. **PowerManager.wakeUp()** — 屏幕唤醒
5. **AOD 90秒** — 官方超时时间

### 修复内容
- `BackScreenController.kt`: 重写，PinReceive → Shizuku widget 两条路径
- 新增 `wakeUpScreen()`、`setScreenTimeout()`、`setBackScreenBrightness()`、`sleepBackScreen()`
- `MainActivity.kt`: 新增 4 个 MethodChannel 方法
- `native_service.dart`: 新增 4 个 Dart 方法
- `home_page.dart`: 自动转发不再要求 Shizuku，转发后自动唤醒+90秒超时
