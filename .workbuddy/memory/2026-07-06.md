# 2026-07-06 工作日志

## 背屏渲染修复（字体大小和位置问题）

### 分析
- 截图（768×520）分析显示：
  - 系统状态栏穿透显示在顶部 y=0~30px
  - 底部导航栏穿透在 y=490~520px
  - 内容区域密集填充 455px，无垂直居中
- 代码审查发现 4 个核心问题

### 修改的文件
- `android/app/src/main/kotlin/.../BackScreenNotificationActivity.kt` — 完整重写

### 修复内容
1. **密度缩放**：字号参数从 Flutter 以 sp 传入，Kotlin 端现在乘以 `resources.displayMetrics.density` 转换为正确的 px 值
2. **沉浸模式**：新增 `setupImmersiveMode()`，使用 `SYSTEM_UI_FLAG_IMMERSIVE_STICKY` + `HIDE_NAVIGATION` + `FULLSCREEN` 彻底隐藏系统栏
3. **垂直居中**：`measureContentHeight()` 预计算内容高度，`maxOf(p, (h - contentHeight) / 2f)` 居中
4. **行间距优化**：内容行间距从 `fontSize + 4px` 改为 `fontSize * 1.45`，更舒适
5. **颜色解析修复**：`parseColorExtra` 现在支持 Flutter 传来的 8 位 ARGB hex
6. **时间戳颜色**：从半透明遮罩改为固定 `#A0A0A0`
7. **配置变更恢复**：`onConfigurationChanged` 和 `onWindowFocusChanged` 时重新设沉浸模式

## 修复通知消除后背屏未关闭 & 返回手势不可用

### 修改的文件
- `android/.../BackScreenNotificationActivity.kt` — 新增 dismiss 机制 + 返回手势处理
- `android/.../BackScreenController.kt` — 重写 `dismissBackScreen()` 不用 force-stop

### 改动
1. **BackScreenNotificationActivity**：
   - `onBackPressed()` 重写 → finish + 启动官方背屏
   - `onNewIntent` 检查 `dismiss=true` → finishAndRestore
   - `onCreate` 检查 `dismiss=true` → 防重建
   - `scheduleAutoDismiss()` 超时后也调用 `finishAndRestore()`
   - `restoreSubscreen()` → `packageManager.getLaunchIntentForPackage("com.xiaomi.subscreencenter")` 启动官方背屏
   - `buildDismissIntent()` 静态方法供 Shizuku 调用

2. **BackScreenController**：
   - `dismissBackScreen()` 改为通过 Shizuku 发送 dismiss intent，不再 force-stop 整个应用
   - 通知监听器不受影响，继续正常工作
