# 2026-07-05

## 修复应用列表权限检测和首次启动引导

### 问题
1. 澎湃OS「应用列表权限」已在系统设置中授予，但 SettingsPage 的「已安装应用」仍显示"无法获取"
2. 首次启动的权限引导页缺少实时反馈

### 根因
1. `PermissionHelper.isInstalledAppsPermissionSupported()` 使用 `resolveContentProvider` → 官方文档要求用 `getPermissionInfo`
2. SettingsPage 仅在 `initState()` 中调用一次，用户从系统设置返回后状态不刷新
3. `getInstalledApps()` 默认 `useShizuku=true`，Shizuku 路径截获调用但 `getApplicationInfo` 因 GET_INSTALLED_APPS 限制全部失败 → 返回空列表
4. MethodChannel `invokeMethod<List<dynamic>>` 泛型与 Kotlin 的 `List<Map>` 不匹配 → result 为 null → 返回 []

### 修复
1. **PermissionHelper.kt**: 按官方文档修正，使用 `getPermissionInfo` 方式
2. **settings_page.dart**: 添加 `WidgetsBindingObserver`，resume 时自动刷新 + stream 监听
3. **permission_guide_page.dart**: 完全重写，实时显示权限状态、进度卡片
4. **MainActivity.getInstalledApps**: 应用列表强制走直接路径，不经过 Shizuku
5. **NativeService.getInstalledApps**: 改泛型为手动类型判断，默认 `useShizuku=false`
6. **BackScreenController.getInstalledApps**: 改用 `ApplicationInfoFlags.of(0L)` 替代 `MATCH_ALL`

## 背屏转发失败根因诊断

创建完整诊断报告 `doc/diagnostic_report_2026-07-05.md`，核心发现：

### Shizuku 不够用的两层原因
1. **SELinux Enforcing**: shell 域无法写 `theme_data_file` 类型的 notification_widget.json，需要 platform_app 域才能绕过
2. **Android 16 受保护广播**: SUB_SCREEN_ON 连 shell (uid=2000) 也不允许发送，仅 platform 签名系统应用有权

### 背屏真实控制机制（与 FBS 当前实现完全不同）
- 背屏是独立副屏，通过 SubScreenDozeService (DreamService/AOD) 渲染
- 唤醒: 双击背屏 → TouchInteractionService.onDoubleTap → PowerManager.wakeUp() 反射
- 休眠: 大面积触摸 / 接近传感器 / SUB_SCREEN_OFF 广播 / 90s 超时
- **input keyevent 26 是电源键，作用于主屏，与背屏无关，且会误锁主屏**

### 当前可行方案
- 唯一可靠交互: PinReceiveActivity (ACTION_SEND text/image)，无需任何权限
- 写 JSON 和发广播在无 root 设备上目前无解
- input keyevent 26 应删除

## V2 背屏转发 — MRSS 风格改造

借鉴 MRSS 直接投屏到 display 1 的思路，完全重写转发机制：

### 背屏参数
- 分辨率: 976×596 (display 1), DPI: 450, 宽高比: 244:149 ≈ 1.637:1
- `am start --display 1` 在设备上验证可行

### 新增文件（4个）
- `lib/models/notification_style.dart` — 样式模型: 字号/颜色/背景/布局/时长, 序列化到 Prefs+Intent extras
- `lib/pages/notification_style_page.dart` — 定制页: 实时预览画布(976:596比例) + 滑块/色板控件
- `BackScreenNotificationActivity.kt` — 原生 Canvas 渲染 Activity, 接收 Intent extras, 自动消失

### 修改文件（5个）
- `AndroidManifest.xml` — 注册 Activity (singleInstance, turnScreenOn, showWhenLocked)
- `BackScreenController.kt` — V2 方法: kill subscreencenter → am start --display 1
- `MainActivity.kt` — displayOnBackScreenV2 handler
- `native_service.dart` — displayOnBackScreenV2() Dart 方法
- `home_page.dart` — V2 转发 + 样式页入口(调色板按钮)

### 核心流程
通知 → Flutter 加载 NotificationStyle → Native kill subscreencenter → am start --display 1 BackScreenNotificationActivity(带样式 extras) → Canvas 渲染 → 自动消失
