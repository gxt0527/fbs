## 修复背屏 AOD 状态被破坏 + 前屏清除后背屏通知残留

### 问题1: 前屏清除后背屏通知残留
**状态**: ✅ 已解决
**根因**: 竟态条件。Flutter 异步回调到达时通知可能已被系统移除，导致已删除通知被重新加入 activeNotifications。
**修复**:
1. `BackScreenController` 新增 `recentlyRemovedKeys` 集合，记录最近被移除的 key
2. `onNotificationAdded` 收到 Flutter 回调时先检查 `recentlyRemovedKeys.contains(key)`，命中则跳过
3. `onNotificationRemoved` 移除 key 时加入 `recentlyRemovedKeys`，5 分钟后自动清理

### 问题2: 背屏 AOD 状态被破坏
**状态**: ⚠️ 部分解决 / 存在系统级限制

**已验证有效的修复**:
- `BackScreenController.doPush()` 不再 `force-stop` subscreencenter，避免杀死 AOD 渲染进程
- 移除 10 秒自动 dismiss，背屏通知随系统通知生命周期同步消失
- 使用 `input -d 1 keyevent KEYCODE_WAKEUP` 只唤醒背屏

**已尝试但失败的方案**:
1. `KEYCODE_SLEEP` / `keyevent 26` → 影响双屏，正面屏幕也被息屏
2. `cmd display power-reset 1` → 只能让背屏回到系统认为的"应有状态"（壁纸时钟亮屏），回不到 AOD 暗屏
3. `cmd display power-off 1; power-reset 1` → 背屏黑屏一下后回到壁纸时钟亮屏
4. AOD 系统 widget 绕过 → `/data/system/theme_magic/...` 目录 shell 无写权限，`miui.intent.action.SUB_SCREEN_ON` 是 protected broadcast
5. 5 秒 AOD 亮度过渡 → 5 秒后 Activity 亮度降低有效，但清除通知后系统壁纸时钟仍为亮屏

**根本原因**: 在 HyperOS 折叠屏上，一旦第三方 Activity 把 display 1 从 AOD/DOZE 拉到 ON，Activity 结束后系统不会自动回到 AOD 暗屏。subscreencenter 需要 `miui.intent.action.SUB_SCREEN_OFF` 等 protected broadcast 才能重新进入 AOD，第三方应用无法发送。

**当前效果**:
- 来通知：背屏亮屏显示通知
- 5 秒后：Activity 降低亮度模拟 AOD 暗屏，继续显示通知内容
- 清除通知：背屏显示系统壁纸时钟（亮屏），不影响正面屏幕

**下一步可能**:
- 需要 root / platform 签名 / 系统级权限才能真正恢复 AOD 暗屏
- 或改为 AOD 时不推送 Activity，仅在背屏已点亮时显示通知

### 设备背屏临时无法点亮
**原因**: 调试过程中多次 `cmd display power-off 1` / `disable-display 1` 把背屏状态打乱。
**恢复命令**:
```bash
adb shell "cmd display enable-display 1 && cmd display power-reset 1 && input -d 1 keyevent KEYCODE_WAKEUP"
```
