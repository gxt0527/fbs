## 超级岛通知 — 最终方案

### 核心发现
**只需要一行代码**，不需要 Shizuku、不需要反射：
```kotlin
notification.extras.putBoolean("android.requestPromotedOngoing", true)
```

系统检测到此 extras 后自动添加 `FLAG_PROMOTED_ONGOING` (262144) 标志，通知进入超级岛。

### 完整条件
1. `POST_PROMOTED_NOTIFICATIONS` 权限（normal 级，任意 app 可声明）
2. Ongoing 通知 + `IMPORTANCE_HIGH` + `CATEGORY_STATUS` + `VISIBILITY_PUBLIC`
3. 在 `nm.notify()` 之前设置 extras

### 探索过程（按时间线）
1. 尝试 `NotificationManager.requestPromotedOngoing(key)` → HyperOS 不存在此方法
2. 尝试 `Builder.setRequestPromotedOngoing(true)` 反射 → hidden API blocked
3. 尝试 `setFlag(FLAG_PROMOTED_ONGOING)` → 被系统剥离
4. 尝试 `INotificationManager.setCanBePromoted()` binder → 需 STATUS_BAR_SERVICE
5. 尝试 Shizuku shell 进程绑 binder → 调用成功但 flags 未变化
6. **监控 Pronto dumpsys** → 发现 `flags=ONGOING_EVENT|PROMOTED_ONGOING`
7. 分析 framework.dex → 发现 `EXTRA_REQUEST_PROMOTED_ONGOING = "android.requestPromotedOngoing"`
8. 在 extras 中设置 → ✅ 成功！
9. 验证 targetSdk=37 同样有效

### Pronto 的真实做法
- 不调用任何 hidden API
- 不依赖 Shizuku（Pronto 虽有 Shizuku 但不用它来设置超级岛）
- 只在 Notification extras 中设置 `android.requestPromotedOngoing=true`

## 背屏通知镜像 — 持久同步

### 需求
背屏通知显示10s就消失被官方背屏接管。目标：背屏通知保持，与前屏通知栏同步，通知清除时才消失，或在背屏点亮状态下用返回手势清除通知。

### 改动文件 (Kotlin)
- **BackScreenController.kt**: 
  - companion 新增 `backScreenActivityAlive`, `isShizukuReady()`, `hasShizukuPermission()`, `buildMirrorLaunchCommand()`, `appendExtraStatic()`
  - 新增 `ACTION_MIRROR_REFRESH`/`ACTION_MIRROR_DISMISS` 广播常量
- **FBSNotificationListenerService.kt**:
  - companion 新增 `instance`, `mirrorEnabled` (SharedPreferences 持久化), `updateMirrorEnabled()`
  - 新增 `mirroredItems: ConcurrentHashMap<String, NotifItem>`
  - `onNotificationPosted` → `feedMirror()` → 自动加入镜像列表 → `pushMirror()`
  - `onNotificationRemoved` → 移除镜像 → `refreshOrDismissMirror()`
  - 新增 `clearAllMirrored()`, `buildNotificationJson()`, `readStylePrefs()` (从 Flutter SharedPreferences 读样式)
  - 导出 `NotifItem` 数据类
- **BackScreenNotificationActivity.kt**:
  - 新增 `isMirror` 标志 + `MirrorReceiver` (BroadcastReceiver)
  - 新增 `applyMirrorUpdate()`, `applyStyleFromBundle()`, `wakeToShow()`, `cancelCurrentMirrorNotification()`
  - `enterAodMode()`: **不清除 FLAG_KEEP_SCREEN_ON**（防止 display 1 睡眠后被 subscreencenter 接管）
  - `dispatchKeyEvent(BACK)`: 镜像模式 → 清除当前前端通知；非镜像 → finishAndRestore
  - `onCreate` 注册动态广播接收者，`onDestroy` 注销
  - 激活时设 `backScreenActivityAlive = true`

### 改动文件 (Dart)
- **MainActivity.kt**: 新增 `isMirrorEnabled`, `setMirrorEnabled`, `clearAllMirroredNotifications` native handlers
- **native_service.dart**: 新增对应 Dart 方法
- **home_page.dart**: 清除背屏按钮改为调用 `clearAllMirroredNotifications()` (取消所有前端镜像通知) → 背屏自动消失
- **settings_page.dart**: 新增「背屏镜像」SwitchListTile 开关

### 架构
```
前端通知栏变化
    ↓
FBSNotificationListenerService (onNotificationPosted / onNotificationRemoved)
    ↓
mirroredItems (ConcurrentHashMap<key, NotifItem>)
    ↓
pushMirror()
    ├─ backScreenActivityAlive=true → sendBroadcast(MIRROR_REFRESH) → Activity.applyMirrorUpdate()
    └─ backScreenActivityAlive=false → Shizuku execShell(am start ...) → launch Activity on display 1

Activity (display 1)
    ← MIRROR_REFRESH 广播: 更新列表+唤醒显示
    ← MIRROR_DISMISS 广播: finishAndRestore (hand back to subscreencenter)
    BACK gesture (镜像模式): cancelNotification(key) → 前端通知取消 → onNotificationRemoved → 刷新/消除
```

### 关键设计决策
- **KEEP_SCREEN_ON 不清除**: 防止 display 1 睡眠后 subscreencenter 接管（原 10s 消失的根因）
- **force-stop subscreencenter**: 启动 Activity 后杀官方背屏，防止 SECONDARY_HOME 抢占 display 1
- **定期 WAKEUP**: 15s 间隔 `input -d 1 keyevent KEYCODE_WAKEUP` 防止 display 1 DOZE（双保险）
- **finish 后恢复 subscreencenter**: 关闭时 `am start SECONDARY_HOME` 恢复官方背屏
- **广播通信**: 同进程内 Activity ←→ ListenerService 用直发广播（设置 package），无需 Shizuku
- **样式从 SharedPreferences 读取**: 镜像通知自动沿用 Flutter 层配置的颜色/字号/布局
- **`清除背屏` 按钮**: 取消所有前端镜像通知 → 自然触发背屏消除 (幂等)

## 背屏通知镜像 — 最终确认可用 (2026-07-08)

### 已验证效果
- ✅ 背屏通知与状态栏同步：通知到达 → 背屏显示；通知清除 → 背屏消失
- ✅ 背屏不会 10s 消失：force-stop subscreencenter + KEEP_SCREEN_ON + screen_off_timeout=90000
- ✅ 清除背屏不会跳到前屏白屏

### 最终修复链

| # | 问题 | 修复 |
|---|------|------|
| 1 | 10s 后被官方背屏接管 | force-stop subscreencenter（移屏同一条命令）+ screen_off_timeout=90000 + 保留 KEEP_SCREEN_ON + 15s 定期 WAKEUP |
| 2 | 清除后跳到 FBS 白屏 | `finishAndRestore()` 去掉 `am start SECONDARY_HOME`，靠系统自动恢复 |
| 3 | `dismissBackScreen` 启动到前屏 | 改用 DISMISS 广播直送 display 1 |
| 4 | 首页 Shizuku 按钮不显示 | 条件从 `running&&!perm` 改为 `!ok`；加 WidgetsBindingObserver 自动刷新 |
| 5 | Service 未在 Manifest 注册 | 补回 `<service>` 声明 |

### 与旧版 old_bsc.kt 的对比
旧版已验证的 `doPush()` 流程: `input keyevent KEYCODE_WAKEUP` → `am start` → `service call task 50 i32 $tid i32 1; am force-stop subscreencenter` → `screen_off_timeout=90000`。当前实现已完全对齐。
