## 超级岛通知 — 最终方案

### 核心发现
**只需要一行代码**，不需要 Shizuku、不需要反射：
```kotlin
notification.extras.putBoolean("android.requestPromotedOngoing", true)
```

系统检测到此 extras 后自动添加 `FLAG_PROMOTED_ONGOING` (262144) 标志，通知进入超级岛。

### 完整条件
1. `POST_PROMOTED_NOTIFICATIONS` 权限（normal 级，任意 app 可声明）
2. Ongoing 通知 + `IMPORTANCE_HIGH` + `CATEGORY_STATUS` + `VISIBILITY_PUBLIC`
3. 在 `nm.notify()` 之前设置 extras

### 探索过程（按时间线）
1. 尝试 `NotificationManager.requestPromotedOngoing(key)` → HyperOS 不存在此方法
2. 尝试 `Builder.setRequestPromotedOngoing(true)` 反射 → hidden API blocked
3. 尝试 `setFlag(FLAG_PROMOTED_ONGOING)` → 被系统剥离
4. 尝试 `INotificationManager.setCanBePromoted()` binder → 需 STATUS_BAR_SERVICE
5. 尝试 Shizuku shell 进程绑 binder → 调用成功但 flags 未变化
6. **监控 Pronto dumpsys** → 发现 `flags=ONGOING_EVENT|PROMOTED_ONGOING`
7. 分析 framework.dex → 发现 `EXTRA_REQUEST_PROMOTED_ONGOING = "android.requestPromotedOngoing"`
8. 在 extras 中设置 → ✅ 成功！
9. 验证 targetSdk=37 同样有效

### Pronto 的真实做法
- 不调用任何 hidden API
- 不依赖 Shizuku（Pronto 虽有 Shizuku 但不用它来设置超级岛）
- 只在 Notification extras 中设置 `android.requestPromotedOngoing=true`
